Green vs Red

This is my documentation in addition to the task implementation in C#. I have implemented the main logic using a custom matrix of nodes. Then I have constructed an inheritance structure of nodes, where the child classes of the Node class are implementing the logic depending on the relevant "color" of the node. Please see the diagram below, and also you can see explanation for each class on the left side bar.

StartUp Class

The static StartUp class contains all the static variables which the application needs to read and pass the user`s input. There is also a separate method for reading and checking the user input - ReadInput(). The ReadInput() method reads, and converts the input and also uses an intance of the InputValidator class. The method passes all the arguments to the validator class for checking. After the input is checked the Main() method creates an instances of the AdjacencyMatrix class and the Engine class. Then it initialize all the input from the static variables to the matrix and runs the Engine.Start() method for receiving the result and printing it to the console as required.

Engine Class

The Engine class receives an injected reference of the AdjacencyMatrix class. Then it runs all the core public methods of the matrix in the Start() method. The class is responsible for running and iterating all the core behaviour of the matrix in order to fetch the requested count of appearances of the green node in the matrix.

Input Validator class

It validates the input passed from the console input. Its behaviour is all based on the task requirements. If the input is incorrect the class throws to the Main() method and InputException with the relevant error message.
