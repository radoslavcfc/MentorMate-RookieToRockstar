Green vs Red

This is my documentation in addition to the task implementation in C#. I have implemented the main logic using a custom matrix of nodes. Then I have constructed an inheritance structure of nodes, where the child classes of the Node class are implementing the logic depending on the relevant "color" of the node. Please see the diagram below, and also you can see explanation for each class on the left side bar.

StartUp Class

The static StartUp class contains all the static variables which the application needs to read and pass the user`s input. There is also a separate method for reading and checking the user input - ReadInput(). The ReadInput() method reads, and converts the input and also uses an intance of the InputValidator class. The method passes all the arguments to the validator class for checking. After the input is checked the Main() method creates an instances of the AdjacencyMatrix class and the Engine class. Then it initialize all the input from the static variables to the matrix and runs the Engine.Start() method for receiving the result and printing it to the console as required.

Engine Class

The Engine class receives an injected reference of the AdjacencyMatrix class. Then it runs all the core public methods of the matrix in the Start() method. The class is responsible for running and iterating all the core behaviour of the matrix in order to fetch the requested count of appearances of the green node in the matrix.

Input Validator class

It validates the input passed from the console input. Its behaviour is all based on the task requirements. If the input is incorrect the class throws to the Main() method and InputException with the relevant error message.

Adjacency Matrix class

This class is a custom implementation of the Adjacency Matrix data structure. It contains a matrix with node elements, as the node`s logic is separated. The properties of the matrix are responsible for the sizes and the collection of the nodes. There is also a property for keeping the addresses of all the node elements which has to be updated. Then the methods are keeping all the logic that is essential for all the operations that could be done over the custom matrix. The program will first book the addresses of the elements that need to be updated and after that it does the updating on a different step. This will protect from element to be changed on the go and change the final result to an incorrect value.

- public void CountGreenNeighbours() This method iterates the matrix and checks the neighbours of each node element. It updates the GreenNeighboursCount for each node on each iteration. In the same iteration the method checks, if the node needs to be updated. If it needs to be updated, its coordinates are booked in the UpdateCoordinates property.
- public void UpdateMatrix() Method for updating the matrix on each iteration. It finds all the nodes that are booked in the UpdateCoordinates property.
- public void InitialNeighboursAllocate() This method is ran only once after the matrix is filled up with its data. This will save neighbour allocation on each iteration.
- private void NeighboursAllocate(Node currentNode, int row, int col) Separated method responsible only for allocating the neighbours which are checked first if they are in the matrix boundaries

INode interface

The interface holds the actual structure of the Node element. It keeps all the members, which needs to be implemented in all the classes that are using this contract.

Abstract Node class

The abstract parent class Node implements the INode interface and its members. It contains the base properties which are typical for the Node element and also keeps abstract members (properties and methods), which are left for the more specific child classes to implement.

Green Node Child class

This child class implements the main Node structure and adds the relevant implementation of the abstract methods. It`s property CurrentValue will return the char value of '1'. The boolean RequireUpdate() method will return true if the count of the green neighbours of the node is equal to 0, 1, 4, 5, 7 or 8. If it is equal to 2, 3, or 6 it will not require an update. Then the Update() method returns a new Node with "colored red" of type RedNode, and will copy the neighbours.

Red Node child class

This child class implements the main Node structure and adds the relevant implementation of the abstract methods. It`s property CurrentValue will return the char value of '0'. The boolean RequireUpdate() method will return true only if the count of the green neighbours of the node is equal to 3 or 6. If the count is any different than the stated one the node will not be updated and will stay the same. Then the Update() method returns a new Node GreenNode, and will copy the neighbours.
